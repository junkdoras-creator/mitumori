# GAS（Google Apps Script）

依頼フローを自動化するスクリプト。Googleフォーム設計書（[../docs/03_Googleフォーム設計.md](../docs/03_Googleフォーム設計.md)）に対応しています。

| ファイル | 役割 |
|---|---|
| `01_createForm.gs` | 依頼フォームを**自動生成**する（要/不要のセクション分岐つき） |
| `02_onFormSubmit_chatwork.gs` | フォーム送信時に**チャットワークへ自動通知**する |

> どちらか一方だけでも使えます。まずは `01` でフォームを作り、運用が回り始めたら `02` で通知を自動化、という順がおすすめです。

## 01. フォーム自動生成

1. [script.google.com](https://script.google.com) で新規プロジェクトを作成し、`01_createForm.gs` を貼り付け。
2. 先頭の `STAFF_LIST`（依頼者の社員名）を自社用に書き換え。
3. `createIraiForm` を実行（初回は承認）。
4. 実行ログに出る **編集URL／回答URL／フォームID** を控える。
5. 回答の蓄積先：フォーム編集画面の［回答］→［スプレッドシートにリンク］で台帳（[../docs/04_台帳設計.md](../docs/04_台帳設計.md)）に接続。

- 全員が同一 Google Workspace なら、`setCollectEmail(true)` のメール自動収集で依頼者を取得できるため、`STAFF_LIST` のプルダウンは外してもOK。
- 「対象税目（複数選択）に応じて自治体欄を出す」分岐はGoogleフォームの仕様上できないため、自治体欄は常時表示＋必須案内にしています（設計書03参照）。

## 02. チャットワーク自動通知

1. `01` と同じプロジェクトに `02_onFormSubmit_chatwork.gs` を貼り付け。
2. ［プロジェクトの設定］→［スクリプト プロパティ］に登録：
   - `CHATWORK_TOKEN` … チャットワークのAPIトークン（[発行方法](https://help.chatwork.com/hc/ja/articles/115000172402)）
   - `CHATWORK_ROOM_ID` … 通知先ルームID（ルームのURL末尾 `rid` の数字）
   - `FORM_ID` … `01` の実行ログに出たフォームID
3. `installChatworkTrigger` を一度だけ実行 → フォーム送信トリガーが作られる。

以降、フォーム送信のたびに該当ルームへ依頼内容が自動投稿されます。投稿には「受託リアクション」「郵送前に最終確認」の注意書きが付きます。

### トラブル時

- 通知が来ない → 実行ログ（［実行数］）でエラー確認。プロパティ未設定が大半です。
- `setCollectEmail` でエラー → Workspace設定によっては手動で「メールアドレスを収集」に変更してください。
