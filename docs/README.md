# 電子申告・製本・郵送 依頼運用 設計書

社員 → 総務パートへ「電子申告データチェック・製本・郵送」を依頼する業務フローの設計書です。
**抜け漏れなし・運用しやすい**ことを目的に、Googleフォーム／スプレッドシート／チャットワークを役割分担させて運用します。

## 全体像

```
[社員] Googleフォームで依頼   →   スプレッドシート台帳   →   チャットワークで通知・受託・最終確認
        （必須化で記入漏れ防止）      （一覧・進捗・完了管理）       （リアクション受託／郵送前の確認）
```

| ツール | 役割 | 防げる抜け漏れ |
|---|---|---|
| Googleフォーム | 依頼の入口。必須項目＋選択式で記入漏れ・表記ゆれを防止 | ① 記入漏れ |
| スプレッドシート | 管理台帳。ステータス・期限・完了報告で進捗を可視化 | ② 埋もれ・③ 完了確認漏れ |
| チャットワーク | 通知・リアクションでの受託・郵送前の最終確認 | 受託の不明確・誤発送 |

## 段階導入（3ステップ）

1. **今すぐ（最小構成）**：チャットの依頼テンプレを定型文化して運用開始＋受託リアクション＋郵送前最終確認。並行して[製本手順書](02_製本手順書.md)を確定。
2. **次の繁忙期前（推奨構成）**：依頼の入口を[Googleフォーム](03_Googleフォーム設計.md)化し、回答を[スプレッドシート台帳](04_台帳設計.md)で進捗・完了管理。
3. **定着後**：フォーム送信時のチャット自動通知（GAS）など自動化を追加。

## まだ決めていない確認ポイント

- **役割分担**：チャットのタスクを「依頼者自身に立てる／パートに立てる」どちらにするか（運用ルールとして要決定）。
- **製本**：束①での税務代理権限証書の位置／地方税メール詳細を都道府県・市町村で分けるか（[02](02_製本手順書.md) の確認ポイント参照）。
