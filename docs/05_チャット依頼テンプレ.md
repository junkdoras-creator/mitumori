# 05. チャットワーク 依頼テンプレ・運用ルール

最小構成（ステップ1）ではこのテンプレをそのまま定型文として運用する。
Googleフォーム導入後（ステップ2）は、フォーム送信を通知し、受託・最終確認のやりとりに使う。

## 受託・最終確認のルール

1. 依頼が投稿されたら、作業に取り掛かるパートが**リアクションで受託**を明確化する。
2. 受託したら、作業前に[受託時チェック項目](01_運用フロー.md#パート受託時チェック項目作業前に必ず確認)を確認する。
3. **郵送前に必ず一度、依頼者（担当者）へ戻して最終確認する**（誤発送防止）。台帳の郵送ステータスは「確認待ち」にする。
4. 担当者OK後に発送し、完了報告（郵送日・追跡番号・控え保管）を台帳へ記録する。

```
・対象事業年度・期：

① 電子申告（要 / 不要）
・期限（日付）：
・対象税目（法人税 / 消費税 / 都道府県 / 市町村）：
・申告先自治体（複数・23区外は必ず明記）：
・添付書類の有無（なし / あり[具体的に：　]）：

② 製本（要 / 不要）
・期限（日付指定[　] / いつでも）：
・特記事項（なし / 元帳添付 / 議事録添付 / その他[　]）：

③ 郵送（要 / 不要）
・期限（日付指定[　] / いつでも）：

・同封物（納付書 / 返送資料 / その他[具体的に：　]）：
```

## フォーム連携（ステップ3・任意）

Google Apps Script のフォーム送信トリガーで、回答内容をチャットワークの該当ルームへ自動投稿できる。
（チャットワークの API トークン／ルームIDが必要。導入時に別途設定する。）
